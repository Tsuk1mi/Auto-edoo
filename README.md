# EDO App - Система электронного документооборота

Полноценное веб-приложение для работы с электронными документами с React, TypeScript, TailwindCSS на фронтенде и Express, MongoDB на бэкенде.

## Структура проекта

Проект состоит из двух основных частей:
- **Фронтенд**: React, TypeScript, TailwindCSS, React Router, Zustand
- **Бэкенд**: Express, TypeScript, MongoDB, JWT аутентификация

## Требования

- Bun 1.0+ или Node.js 18+
- MongoDB (можно использовать Docker)

## Установка и запуск

### Разработка

1. Клонировать репозиторий:
```bash
git clone https://github.com/yourusername/edo-app.git
cd edo-app
```

2. Установить зависимости:
```bash
bun install
cd server && bun install
cd ..
```

3. Запустить MongoDB:
```bash
docker-compose up -d mongodb
```

4. Запустить бэкенд и фронтенд:
```bash
bun run dev:all
```

### С использованием Docker

Для запуска всего приложения с помощью Docker:

```bash
docker-compose up -d
```

## API Endpoints

### Авторизация

- **POST /api/auth/register** - Регистрация нового пользователя
- **POST /api/auth/login** - Авторизация
- **GET /api/auth/me** - Получение данных текущего пользователя (требует авторизации)

### Документы

- **GET /api/documents** - Получение всех документов пользователя (требует авторизации)
- **GET /api/documents/:id** - Получение документа по ID (требует авторизации)
- **POST /api/documents** - Создание нового документа (требует авторизации)
- **PUT /api/documents/:id** - Обновление документа (требует авторизации)
- **DELETE /api/documents/:id** - Удаление документа (требует авторизации)

# Система электронного документооборота (фронтенд)

Фронтенд часть системы электронного документооборота.

## Технологии

- React
- TypeScript
- Vite
- Tailwind CSS
- Zustand (управление состоянием)
- React Router (маршрутизация)
- Axios (HTTP-клиент)

## Установка

1. Склонируйте репозиторий:
```
git clone <ссылка на репозиторий>
```

2. Перейдите в директорию проекта:
```
cd edo-app
```

3. Установите зависимости:
```
npm install
```

4. Настройте переменные окружения:
```
cp .env.example .env
```
Отредактируйте файл `.env` в соответствии с вашей конфигурацией:
```
VITE_API_URL=http://localhost:5000/api
```

## Запуск

### Режим разработки
```
npm run dev
```

### Сборка для production
```
npm run build
```

### Предпросмотр сборки
```
npm run preview
```

## Функциональность

- Авторизация и регистрация пользователей
- Управление документами (создание, редактирование, просмотр, удаление)
- Интеграция с внешними системами
- Мониторинг и просмотр логов системы
- Настройки приложения и профиля пользователя

## Структура проекта

```
src/
  ├── api/              # API-клиент и взаимодействие с сервером
  ├── components/       # Переиспользуемые компоненты
  │   ├── layout/       # Компоненты макета (Header, Sidebar, etc.)
  │   └── ui/           # UI компоненты (Button, Input, Modal, etc.)
  ├── features/         # Функциональные модули
  │   ├── auth/         # Аутентификация
  │   ├── documents/    # Работа с документами
  │   ├── integration/  # Интеграции
  │   └── settings/     # Настройки
  ├── hooks/            # Пользовательские React-хуки
  ├── pages/            # Страницы приложения
  ├── router/           # Настройка маршрутов
  ├── store/            # Хранилище состояния (Zustand)
  ├── types/            # TypeScript типы и интерфейсы
  └── utils/            # Утилиты и вспомогательные функции
```

## Система логирования

Система включает расширенные возможности логирования:

- Консольное логирование (в режиме разработки)
- Отправка критических ошибок на сервер
- Сохранение логов в localStorage (в режиме разработки)
- Просмотр логов через интерфейс приложения (в разделе "Настройки")

## Аутентификация

Аутентификация реализована с использованием JWT-токенов:

- Токен хранится в localStorage
- Защищенные маршруты требуют авторизации
- Автоматическое перенаправление на страницу входа при истечении токена
- Перенаправление обратно на запрошенную страницу после успешного входа

## Интеграция с бэкендом

Требуется запущенный бэкенд-сервер. Настройте URL API в файле .env:
```
VITE_API_URL=http://localhost:5000/api
```

# Система электронного документооборота (EDO)

Эта система предназначена для управления электронными документами, автоматизации рабочих процессов и интеграции с внешними системами.

## Требования

Для запуска проекта вам потребуется:

- [Node.js](https://nodejs.org/) 18+ или [Bun](https://bun.sh/)
- [MongoDB](https://www.mongodb.com/) (опционально, если не используете режим разработки без базы данных)

## Установка и запуск

### Клонирование репозитория

```bash
git clone [url_репозитория]
cd edo-project
```

### Установка зависимостей

С использованием Bun (рекомендуется):

```bash
bun install
cd server && bun install && cd ..
```

Или с использованием npm:

```bash
npm install
cd server && npm install && cd ..
```

### Запуск в режиме разработки

Для запуска только фронтенда:

```bash
bun run dev
```

Для запуска сервера (без подключения к базе данных):

```bash
bun run server
```

Для одновременного запуска фронтенда и сервера:

```bash
bun run dev:all
```

После запуска:
- Фронтенд будет доступен по адресу: http://localhost:5173
- Сервер API будет доступен по адресу: http://localhost:5000

### Демонстрационные данные

В режиме разработки без БД вы можете войти в систему, используя следующие учетные данные:

- **Администратор**:
  - Email: admin@example.com
  - Пароль: password123

- **Обычный пользователь**:
  - Email: user@example.com
  - Пароль: password123

Более подробная информация о демо-данных доступна в файле [DEMO.md](./DEMO.md).

### Запуск с MongoDB

Если вы хотите использовать MongoDB:

1. Установите и запустите MongoDB на вашей машине
2. Измените значение `NODE_ENV` в файле `server/.env` на `development`
3. Запустите сервер командой:
```bash
cd server && bun run dev
```
